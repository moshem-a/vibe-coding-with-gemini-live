The big idea

Most AI demos are text in → text out.
This is voice in → product out.

The model isn't behind a chat window. It holds tool-calls on the Live API that drive React state directly: propose an architecture, write a file, swap a logo, navigate the preview, consult another agent. You speak — the UI changes.
The flywheel: microphone → 16 kHz PCM → Live WebSocket → model thinks → speaks back (24 kHz audio) and emits a function-call → React handler mutates state → UI re-renders → next user utterance. All round-trip in well under a second.

No bundler. No backend round-trip. A single Nginx container serves static React + Babel and talks WebSocket to the model — every other capability is a tool-call routed from React state.
The contract

Gemini Live = audio in, audio out, + tool-calls.

ToolPurpose
propose_architectureEmit a Mermaid GCP flowchart version
approve_architectureMark a version as the chosen one
start_buildFlip the UI into the "writing UI…" splash
write_file / update_fileCreate or replace files; each write is snapshotted for history
show_previewMount the iframe with the generated index.html
generate_logoReturn a placeholder token; real image is generated and swapped at render time
navigate_previewpostMessage into the iframe to switch SPA views by voice
consult_designer / qa / developerFan out to specialist sub-agents (gemini-3.1-pro-preview with role prompts)

Q

QA

Reviews HTML for missing elements, broken interactions, a11y issues, UX gaps. Returns {issues, ok}.
Gates show_preview
E

Developer

Engineering judgment calls: state machines, fragile patterns, cross-browser risks. Returns {approach, code_snippet, considerations, risks}.
For non-trivial JS

Optional · Admin observability

Same agents, on Vertex AI Agent Engine.

1

Build with Google ADK

LlmAgent(name="designer", model="gemini-3.1-pro-preview", instruction=DESIGNER_PROMPT) — same prompt, but now a deployable artifact.
2

Deploy via Cloud Build

No pip in Firebase Studio? No problem. gcloud builds submit agents/ runs the deploy inside python:3.12, writes deployed_engines.json for the proxy image.
3

Proxy on Cloud Run

FastAPI service, ADC creds, fans out to agent_engines.get(...).stream_query(...), streams events back as JSON for the browser.
4

Watch sessions in the console

Each consult call shows up as a session with user_id, message, full trace tree. The agent activity panel deep-links each card to its Agent Engine session.
Inside generate_logo

Tokens over the wire. Bytes at render time.

Gemini Live shouldn't round-trip megabytes of base64 PNG. Instead, generate_logo returns a tiny token string like __ASSET_LOGO__. The model writes that as the <img src>. The host swaps the real data URL in when it builds the iframe srcdoc.

Nano Banana Pro

nano-banana-pro-preview:generateContent · responseModalities: ["IMAGE"]. Fast, iconic, great for logos and stylized art.

Imagen 4 Fast

imagen-4.0-fast-generate-001:predict · aspectRatio: "1:1". Photo-real, great for hero shots and product imagery.
Last-resort: any <img> that fails to load (dead URL, unsubstituted placeholder) auto-swaps to picsum.photos/seed/<alt>/W/H — so the prototype never shows a broken image on stage.
Talking to the running app

Voice navigates the preview. Without rewriting code.

Generated apps are required to be multi-view SPAs: every screen is a <section data-view="..."> and a global window.showView(name) swaps them. 
// In every generated index.html:
window.addEventListener('message', (e) => {
  if (e.data?.type === 'navigate') window.showView(e.data.view, e.data.params);
  else if (e.data?.type === 'back') window.goBack();
});

// On the host: navigate_preview tool-call →
previewIframe.contentWindow.postMessage(
  { type: 'navigate', view: 'settings' }, '*'
);
Say "show me the dashboard." Gemi emits navigate_preview({view: "dashboard"}). The host postMessages into the iframe. Zero re-renders, no file rewrites, sub-100ms.
Final polish

What changed since last week.

1

Hebrew-locked voice

Even if you speak English, Gemi always replies in Hebrew. languageCode: "he-IL" on the Live API + a hard rule in the system prompt.
2

Transcript hidden by default

Auto-transcripts are noisy on stage. The Pair-with-Gemi panel collapses them under one click.
3

Projects auto-save

Every generated project is named, dated, and persisted to localStorage. The "Projects" button in the AppBar loads them back instantly.
4

More speech triggers

Gemi MUST announce "האפליקציה שלך מוכנה!" after show_preview and confirm every update_file — no more silent gaps.
